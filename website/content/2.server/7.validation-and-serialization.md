mion uses `@mionkit/run-types` for automatic validation and serialization.
This is a powerful type system that extracts TypeScript type metadata at compile time and uses it at runtime. 
No need to manage schemas or write extra code for validation and serialization.

## Automatic Validation

All route and hook parameters are automatically validated before execution. If validation fails, an `RpcError` is thrown with details about the validation errors.

<code-import path="packages/examples/src/router/overview-runtypes-example.ts" lang="ts" />

If an invalid request is sent:

```json
// Request with invalid data
{
    "name": "John",
    "email": 123,  // Should be string
    "age": "twenty"  // Should be number
}
```

The response will contain validation errors:

```json
{
    "createUser": {
        "error": {
            "statusCode": 400,
            "message": "Validation Error",
            "type": "validation-error",
            "data": {
                "errors": [
                    {"path": "email", "expected": "string", "received": "number"},
                    {"path": "age", "expected": "number", "received": "string"}
                ]
            }
        }
    }
}
```

## Binary Serialization

For applications where data transfer efficiency is critical, mion supports binary serialization which provides more compact data transfer compared to JSON.
To enable binary serialization for all routes, set the `serialize` option to `'binary'` when initializing the router:

::tip
Binary serialization is especially effective for transferring data structures that contain numeric data. Using the numeric type formats from `@mionkit/type-formats` (like `NumInt32`, `NumUInt8`, `NumFloat`) allows the binary serializer to use optimized fixed-size encodings, significantly reducing payload sizes compared to JSON.
::

<code-import path="packages/examples/src/router/binary-server-example.ts" lang="ts" />

### Setting Binary Serialization Per Route

You can also enable binary serialization for specific routes by setting the `serializer` option:

```ts
const routes = {
    // JSON serialization (default)
    jsonRoute: route((ctx, data: MyType) => data),

    // Binary serialization for this route
    binaryRoute: route((ctx, data: MyType) => data, {serializer: 'binary'}),
} satisfies Routes;
```

## Supported Types

mion supports a wide range of TypeScript types out of the box:

- **Primitives**: `string`, `number`, `boolean`, `null`, `undefined`
- **Objects**: Plain objects, interfaces, type aliases
- **Arrays**: `Array<T>`, `T[]`, tuples
- **Special types**: `Date`, `BigInt`, `Map`, `Set`
- **Union types**: `string | number`, discriminated unions
- **Optional properties**: `property?: Type`
- **Readonly properties**: `readonly property: Type`
