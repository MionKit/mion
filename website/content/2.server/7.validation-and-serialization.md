mion uses `@mionkit/run-types` for automatic validation and serialization.
This is a powerful type system that extracts TypeScript type metadata at compile time and uses it at runtime. 
No need to manage schemas or write extra code for validation and serialization.

## Automatic Validation

All route and hook parameters are automatically validated before execution. If validation fails, an `RpcError` is thrown with details about the validation errors.

<code-import ts="0" path="packages/router/examples/validation-auto.ts" lang="ts" />

If an invalid request is sent:

```json
// Request with invalid data
{
    "name": "John",
    "email": 123,  // Should be string
    "age": "twenty"  // Should be number
}
```

The response will contain validation errors:

```json
{
    "createUser": {
        "error": {
            "statusCode": 400,
            "message": "Validation Error",
            "type": "validation-error",
            "data": {
                "errors": [
                    {"path": "email", "expected": "string", "received": "number"},
                    {"path": "age", "expected": "number", "received": "string"}
                ]
            }
        }
    }
}
```

## Supported Types

mion supports a wide range of TypeScript types out of the box:

- **Primitives**: `string`, `number`, `boolean`, `null`, `undefined`
- **Objects**: Plain objects, interfaces, type aliases
- **Arrays**: `Array<T>`, `T[]`, tuples
- **Special types**: `Date`, `BigInt`, `Map`, `Set`
- **Union types**: `string | number`, discriminated unions
- **Optional properties**: `property?: Type`
- **Readonly properties**: `readonly property: Type`
