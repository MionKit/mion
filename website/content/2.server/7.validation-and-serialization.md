title: Validation & Serialization

mion uses `@mionkit/run-types` for automatic validation and serialization. This is a powerful type system that extracts TypeScript type metadata at compile time and uses it at runtime.

## Automatic Validation

All route and hook parameters are automatically validated before execution. If validation fails, an `RpcError` is thrown with details about the validation errors.

```ts
import {Routes, route} from '@mionkit/router';

interface User {
    name: string;
    email: string;
    age: number;
}

const routes = {
    createUser: route((ctx, user: User) => {
        // user is guaranteed to be valid at this point
        return saveUser(user);
    }),
} satisfies Routes;
```

If an invalid request is sent:

```json
// Request with invalid data
{
    "name": "John",
    "email": 123,  // Should be string
    "age": "twenty"  // Should be number
}
```

The response will contain validation errors:

```json
{
    "createUser": {
        "error": {
            "statusCode": 400,
            "message": "Validation Error",
            "type": "validation-error",
            "data": {
                "errors": [
                    {"path": "email", "expected": "string", "received": "number"},
                    {"path": "age", "expected": "number", "received": "string"}
                ]
            }
        }
    }
}
```

## Automatic Serialization

Dates, BigInts, and other complex types are automatically serialized and deserialized:

```ts
import {Routes, route} from '@mionkit/router';

interface Order {
    id: string;
    createdAt: Date;  // Automatically serialized/deserialized
    total: bigint;    // Automatically serialized/deserialized
}

const routes = {
    getOrder: route((ctx, id: string): Order => {
        return {
            id,
            createdAt: new Date(),
            total: 1000n,
        };
    }),
} satisfies Routes;
```

::note
Serialization works transparently - you work with native TypeScript types and mion handles the JSON conversion automatically.
::

## Supported Types

mion supports a wide range of TypeScript types out of the box:

- **Primitives**: `string`, `number`, `boolean`, `null`, `undefined`
- **Objects**: Plain objects, interfaces, type aliases
- **Arrays**: `Array<T>`, `T[]`, tuples
- **Special types**: `Date`, `BigInt`, `Map`, `Set`
- **Union types**: `string | number`, discriminated unions
- **Optional properties**: `property?: Type`
- **Readonly properties**: `readonly property: Type`

## Client-Side Validation

The mion client can validate parameters locally before sending requests:

```ts
import {initClient} from '@mionkit/client';
import type {MyApi} from './server.routes';

const {routes} = initClient<MyApi>({baseURL: 'http://localhost:3000'});

// Validate without making a request
const errors = await routes.createUser({
    name: 'John',
    email: 123,  // Invalid
    age: 'twenty',  // Invalid
}).typeErrors();

console.log(errors);
// [
//   {path: 'email', expected: 'string', received: 'number'},
//   {path: 'age', expected: 'number', received: 'string'}
// ]
```

This allows you to catch validation errors before making network requests, improving user experience and reducing server load.
