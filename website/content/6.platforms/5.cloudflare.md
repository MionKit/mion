mion can run on Cloudflare Workers for globally-distributed edge deployments. The `@mionjs/platform-cloudflare` package provides a Web standard `Request -> Response` fetch handler with access to Worker bindings via the platform context.

## Installation

```bash
npm install @mionjs/platform-cloudflare @mionjs/router
```

## wrangler.toml Example

A minimal `wrangler.toml` for a mion Worker:

```toml
name = "my-mion-api"
main = "dist/worker.js"
compatibility_date = "2024-01-01"
```

The `main` field points to the bundled output produced by Vite (see [Vite Configuration](#vite-configuration) below). Run `wrangler deploy` to publish.

## Vite Configuration

::caution
Cloudflare Workers do not allow `new Function()` or `eval()`. mion never generates code at runtime on the edge: the [mion Vite plugin](/devtools/vite-config) compiles every route's validation and serialization functions at **build time** and injects them straight into the bundle, so no special AOT step is required.
::

A minimal `vite.config.ts` for a Cloudflare Worker:

```ts [vite.config.ts]
import {defineConfig} from 'vite';
import {resolve} from 'path';
import {mionVitePlugin} from '@mionjs/devtools/vite-plugin';

export default defineConfig({
    plugins: [
        mionVitePlugin({
            runTypes: {tsConfig: resolve(__dirname, 'tsconfig.json')},
        }),
    ],
    build: {
        lib: {
            entry: 'src/index.ts',
            formats: ['es'],
            fileName: 'worker',
        },
        rollupOptions: {
            output: {inlineDynamicImports: true},
        },
    },
});
```

`inlineDynamicImports: true` is required because Workers don't support dynamic code splitting â€” every dependency must be bundled into a single file.

The Worker entry itself needs no special setup â€” it just registers the routes with `initMionRouter` and exports the fetch handler, exactly as shown in [Quick Start](#quick-start) above. The plugin has already replaced each route's validators with precompiled functions at build time, so there is no AOT flag, cache import, or virtual module to wire up.

:spacer
