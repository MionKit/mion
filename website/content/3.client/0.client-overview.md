## Features

- ✅ Strongly typed APIs with autocompletion and static type checking.
- ✅ Fully typed list of remote methods with its parameters and return values.
- ✅ Result pattern with 4-tuple `[routeResult, routeError, hooksResults, hooksErrors]` - never throws, always returns.
- ✅ Automatic Validation and Serialization out of the box.
- ✅ Local Validation (no need to make a server request to validate parameters)
- ✅ Prefill request data to persist across multiple calls.

### Calling Routes

The `routes` and `hooks` objects returned from `initClient` contain all remote routes and hooks.
If a hook is not public (does not expect any parameters and can't return data) then it is not included within the `hooks` object.

Calling routes in the client generates a `RouteSubRequest`. We use the `call()` method to perform the remote call.
The `call()` method returns a 4-tuple `[routeResult, routeError, hooksResults, hooksErrors]` - it **never throws**.

### Prefilling Hooks data

For cases like authorization hooks that are required for all requests, we can use the `prefill()` method of `HookSubRequest`. This will automatically prefill any future route calls that require that hook.

When using prefilled hooks, hook results and errors are available in **two ways**:
1. **TypedEvent handlers** - Strongly typed by errors
2. **tuple results** - Generic `RpcError` types

<code-import path="packages/examples/src/client/client-prefill-hooks.ts" lang="ts" />

::note
**TypedEvent handlers** are called after **every** API request that includes the hook.
::

::tip
Use **TypedEvent handlers** for global error handling with prefilled hooks. They provide **strongly typed** error handling based on the error `type` string.
::
