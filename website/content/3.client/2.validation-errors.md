When validation fails, mion returns an`RpcError<'validation-error', ValidationErrorData>` containing technical information about the failure. The `getFriendlyErrors` function transforms these errors into user-friendly messages that can be displayed directly in your UI.

## Defining Friendly Error Handlers

The `FriendlyErrors<T>` type creates a map that mirrors your data structure. Each property can have a handler function that receives error parameters and returns a friendly message string.

For nested objects, you can define nested handler objects. For arrays, a single handler applies to all array elements.
::

## Error Parameter Types

Unfortunately the type of Formats Params are not automatically inferred. To get proper intellisense, you need to narrow down the error parameter types using specific `ErrorParams` types. These types provide intellisense for the format parameters that might have failed.

### Available Error Parameter Types

| Type | Use For |
|------|---------|
| `StringErrorParams` | `StrFormat` |
| `EmailErrorParams` | `StrEmail` | 
| `UrlErrorParams` | `StrUrl` | 
| `DomainErrorParams` | `StrDomain` | 
| `IPErrorParams` | `StrIP` |
| `UUIDErrorParams` | `StrUUID` | 
| `DateTimeErrorParams` | `StrDateTime` | 
| `DateErrorParams` | `StrDate` | 
| `TimeErrorParams` | `StrTime` | 
| `NumberErrorParams` | `NumFormat` | 
| `BigIntErrorParams` | `BigNumFormat` | 

All error parameter types also include:
- `rtError` - The full `RunTypeError` object for access to `expected`, `path`, etc.
- `propName` - The name of the property that failed validation
- `index` - The array index (for array element errors)

## Important Considerations

### Handlers May Be Called Multiple Times

A single property can fail multiple validation constraints. For example, a string might fail both `minLength` and `pattern` validations. The handler function is called **once per error**, not once per property.

The result object contains `string[]` arrays because multiple errors can occur for the same property. Duplicate messages are automatically removed.

### Accessing Error Values

Each error parameter (like `minLength`, `max`, etc.) is a `TypeFormatError` object containing:
- `val` - The constraint value that was violated
- `name` - The format name
- `formatPath` - Path to the format parameter

<code-import path="packages/examples/src/client/friendly-errors-others.ts" lang="ts" commentStart="start-error-values" commentEnd="end-error-values" />

### Default Error Printer

If no handler is provided for a property, `getFriendlyErrors` uses a default error printer and logs a warning to the console. The default messages are technical and not suitable for UI and end users.

Always provide custom handlers for user-facing applications.

::warning
Default messages look like: `name: minLength validation failed (expected 5)` - these are not user-friendly!
::
