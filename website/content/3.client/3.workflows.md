## Workflow vs Single Route Call

The key difference between `call()` and `workflow()` is in the response structure:

| Aspect | `route.call()` | `workflow([routes...])` |
|--------|----------------|-------------------------|
| **HTTP Requests** | One per route | One for all routes |
| **Result** | Single value | Array of values |
| **Error** | Single error | Array of errors |
| **Position** | Direct access | Index-based access |

<code-import path="packages/examples/src/client/workflow-vs-single.ts" lang="ts" />

### Response Pattern Comparison

```ts
// Single route call - direct result/error
const [result, error, linkedFnResults, linkedFnErrors] = await route.call();

// Workflow - arrays of results/errors
const [[result1, result2], [error1, error2], linkedFnResults, linkedFnErrors] = await workflow([route1, route2]);
```

::tip
**When to use workflow:**
- Fetching data for a page that needs multiple API calls
- Batch operations that don't depend on each other
- Reducing network latency in high-latency environments
::

## Error Handling in Workflows

In a workflow, each route can succeed or fail independently. The error array contains errors at the same index as their corresponding routes:

```ts
const [[user, order, sum], [userError, orderError, sumError]] = await workflow([
    routes.users.getById('USER-123'),    // index 0
    routes.orders.getById('ORDER-404'),  // index 1 - this will fail
    routes.utils.sum(5, 2),              // index 2
]);

// Results:
// user = {id: 'USER-123', ...}  - success
// userError = undefined

// order = undefined              - failed
// orderError = RpcError<'order-not-found'>

// sum = 7                        - success
// sumError = undefined
```

::warning
A failing route does NOT stop other routes from executing. Each route in the workflow is processed independently.
::

## Serialization in Workflows

Complex types like `Date`, `Map`, and `Set` are automatically serialized and deserialized in workflows, just like single route calls:

```ts
const testDate = new Date('2024-06-15T12:30:00.000Z');
const testMap = new Map([['a', 1], ['b', 2]]);

const [[date, map], [dateError, mapError]] = await workflow([
    routes.getSameDate(testDate),
    routes.getSameMap(testMap),
]);

// date is a Date instance
// map is a Map instance
```

## Constraints

::stylish-list
---
type: x
---
- [Same client instance:]{.text-highlighted} All routes in a workflow must use the same client
- [At least one route:]{.text-highlighted} Workflow requires at least one route subrequest
::

```ts
// ❌ This will throw an error
const {routes: routes1} = initClient<MyApi>({baseURL: 'http://server1.com'});
const {routes: routes2} = initClient<MyApi>({baseURL: 'http://server2.com'});

await workflow([routes1.getUser('1'), routes2.getOrder('1')]); // Error!

// ✅ Use routes from the same client
const {routes} = initClient<MyApi>({baseURL: 'http://localhost:3000'});
await workflow([routes.getUser('1'), routes.getOrder('1')]); // Works!
```
