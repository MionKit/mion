title: Error Handling
description: Learn how to handle errors in mion APIs.

mion uses `RpcError` to handle all errors in routes and hooks. The `RpcError` class provides a consistent way to throw and handle errors across your API.

## Throwing Errors

You can throw errors in routes and hooks using the `RpcError` class:

```ts
import {RpcError} from '@mionkit/core';
import {Routes, route} from '@mionkit/router';

const routes = {
    getUser: route((ctx, userId: string) => {
        const user = findUser(userId);
        if (!user) {
            throw new RpcError({
                statusCode: 404,
                publicMessage: 'User not found',
                type: 'user-not-found',
            });
        }
        return user;
    }),
} satisfies Routes;
```

## Error Response Format

When an error is thrown, the response will contain the error details:

```json
{
    "getUser": {
        "error": {
            "statusCode": 404,
            "message": "User not found",
            "type": "user-not-found"
        }
    }
}
```

::note
Only `publicMessage` is sent to the client. The `message` property is for internal logging only and is never exposed to clients.
::

## Handling Multiple Errors

When multiple hooks or routes throw errors, all errors are collected and returned in the response:

```ts
import {RpcError} from '@mionkit/core';
import {Routes, hook, route} from '@mionkit/router';

const routes = {
    validateInput: hook((ctx, input: string) => {
        if (!input) {
            throw new RpcError({
                statusCode: 400,
                publicMessage: 'Input is required',
                type: 'validation-error',
            });
        }
    }),
    processData: route((ctx, data: string) => {
        // This won't execute if validateInput throws
        return `Processed: ${data}`;
    }),
} satisfies Routes;
```

## Run On Error

Hooks with `runOnError: true` will execute even when previous hooks or routes throw errors. This is useful for logging or cleanup:

```ts
import {Routes, hook} from '@mionkit/router';

const routes = {
    logger: hook(
        (ctx) => {
            const errors = ctx.request.internalErrors;
            if (errors.length > 0) {
                console.error('Request errors:', errors);
            }
        },
        {runOnError: true}
    ),
    // ... other routes
} satisfies Routes;
```

## Type Reference

### RpcError

```ts
export class RpcError extends Error {
    /** HTTP status code */
    statusCode: number;
    /** Public message sent to client */
    publicMessage: string;
    /** Error type identifier */
    type: string;
    /** Internal message for logging (not sent to client) */
    message: string;
    /** Optional error data */
    data?: unknown;
}
```

### ErrorResponse

```ts
export interface ErrorResponse {
    statusCode: number;
    message: string;
    type: string;
    data?: unknown;
}
```
