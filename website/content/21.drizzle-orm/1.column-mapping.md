---
title: Column Mapping
description: How types are mapped to database columns when using mion Drizzle ORM package.
toc: false
---

The `@mionjs/drizzle` package automatically maps TypeScript types to appropriate Drizzle ORM column definitions. This mapping is database-aware, using native column types where available (e.g., PostgreSQL's `uuid()` and `inet()`) and falling back to compatible alternatives for databases with fewer native types (e.g., SQLite uses `text()` for most string formats).

Type formats from `@mionjs/type-formats` provide additional semantic information that enables more precise column mapping. For example, `FormatInteger` maps to an integer column rather than a floating-point column, and `FormatEmail` uses the appropriate varchar length for email addresses.

## Primitive Types

| TypeScript Type | PostgreSQL | MySQL | SQLite |
|-----------------|------------|-------|--------|
| `string` | `varchar(255)` | `varchar(255)` | `text()` |
| `number` | `doublePrecision()` | `double()` | `real()` |
| `boolean` | `boolean()` | `boolean()` | `integer({mode: 'boolean'})` |
| `bigint` | `bigint({mode: 'bigint'})` | `bigint({mode: 'bigint'})` | `blob({mode: 'bigint'})` |
| `Date` | `timestamp()` | `timestamp()` | `integer({mode: 'timestamp'})` |

## String Format Types

| Format Type | PostgreSQL | MySQL | SQLite |
|-------------|------------|-------|--------|
| `FormatUUID` / `FormatUUIDv7` | `uuid()` | `varchar(36)` | `text()` |
| `FormatEmail` | `varchar(254)` | `varchar(254)` | `text()` |
| `StrURL` | `varchar(2048)` | `varchar(2048)` | `text()` |
| `FormatDomain` | `varchar(253)` | `varchar(253)` | `text()` |
| `FormatIP` / `FormatIPv4` / `FormatIPv6` | `inet()` | `varchar(45)` | `text()` |
| `FormatStringDateTime` | `timestamp()` | `datetime()` | `text()` |
| `FormatStringDate` | `date()` | `date()` | `text()` |
| `FormatStringTime` | `time()` | `time()` | `text()` |
| `FormatString` (with maxLength) | `varchar(maxLength * buffer)` | `varchar(maxLength * buffer)` | `text()` |

Well-known formats like `FormatEmail`, `StrURL`, and `FormatDomain` use their standard maximum lengths without any buffer applied.

## Number Format Types

All number formats with the `integer: true` parameter map to integer columns, while formats with `float: true` or no integer constraint map to floating-point columns.

| Format Type | Description | PostgreSQL | MySQL | SQLite |
|-------------|-------------|------------|-------|--------|
| `FormatInteger` | Generic integer | `integer()` | `int()` | `integer()` |
| `FormatFloat` | Generic float | `doublePrecision()` | `double()` | `real()` |
| `FormatPositive` | Number >= 0 | `doublePrecision()` | `double()` | `real()` |
| `FormatNegative` | Number <= 0 | `doublePrecision()` | `double()` | `real()` |
| `FormatPositiveInt` | Integer >= 0 | `integer()` | `int()` | `integer()` |
| `FormatNegativeInt` | Integer <= 0 | `integer()` | `int()` | `integer()` |
| `FormatInt8` | 8-bit signed (-128 to 127) | `integer()` | `int()` | `integer()` |
| `FormatInt16` | 16-bit signed (-32768 to 32767) | `integer()` | `int()` | `integer()` |
| `FormatInt32` | 32-bit signed | `integer()` | `int()` | `integer()` |
| `FormatUInt8` | 8-bit unsigned (0 to 255) | `integer()` | `int()` | `integer()` |
| `FormatUInt16` | 16-bit unsigned (0 to 65535) | `integer()` | `int()` | `integer()` |
| `FormatUInt32` | 32-bit unsigned (0 to 4294967295) | `integer()` | `int()` | `integer()` |

::callout{type="info"}
While the database column type is the same for all integer formats, the type constraints (min/max values) are still validated at runtime by mion's type system.
::

## BigInt Format Types

| Format Type | PostgreSQL | MySQL | SQLite |
|-------------|------------|-------|--------|
| `FormatBigInt` | `bigint({mode: 'bigint'})` | `bigint({mode: 'bigint'})` | `blob({mode: 'bigint'})` |

## Complex Types

| TypeScript Type | PostgreSQL | MySQL | SQLite |
|-----------------|------------|-------|--------|
| `T[]` (array) | `jsonb()` | `json()` | `text({mode: 'json'})` |
| `{...}` (object) | `jsonb()` | `json()` | `text({mode: 'json'})` |
