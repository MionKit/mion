Once you've generated AOT caches, you need to load them in your application. This page explains how to use AOT caches in both server and client contexts.

## Server Usage

Loading AOT caches on the server is straightforward. Import and use the cache exports from your AOT package before initializing the router.

### Loading AOT Caches

```ts
// Import cache exports from your AOT package
import {routerCache, jitFnsCache, pureFnsCache} from 'my-api-aot';

// Import the cache loading function from @mionkit/core
import {addAOTCaches} from '@mionkit/core';

// Load the pre-compiled caches BEFORE initializing the router
addAOTCaches(jitFnsCache, pureFnsCache);

// Now initialize your router - it will use the pre-compiled functions
import {initMionRouter} from '@mionkit/router';
import {myRoutes} from './routes';

export const myApi = initMionRouter(myRoutes);
```

::note
**Important:** You must call `addAOTCaches()` before initializing the router. The router checks for pre-compiled functions during initialization and uses them instead of generating new ones.
::

### Complete Server Example

:::code-group
```ts [server.ts]
// Load AOT caches first
import {routerCache, jitFnsCache, pureFnsCache} from 'my-api-aot';
import {addAOTCaches} from '@mionkit/core';

addAOTCaches(jitFnsCache, pureFnsCache);

// Then initialize router and server
import {initMionRouter} from '@mionkit/router';
import {initHttp} from '@mionkit/http';
import {routes} from './routes';

export const myApi = initMionRouter(routes);

initHttp({port: 3000});
console.log('Server running on port 3000');
```

```ts [routes.ts]
import {Routes, route} from '@mionkit/router';

export interface User {
    id: string;
    name: string;
    email: string;
}

export const routes = {
    users: {
        getById: route((ctx, id: string): User => {
            return {id, name: 'John', email: 'john@example.com'};
        }),
        create: route((ctx, user: Omit<User, 'id'>): User => {
            return {id: 'USER-123', ...user};
        }),
    },
} satisfies Routes;
```
:::

## Client Usage

The mion client automatically loads AOT caches from `@mionkit/aot-caches` on initialization. For custom AOT caches, you can load them manually.

### Using Default AOT Caches

The client automatically uses the default mion AOT caches:

```ts
import {initClient} from '@mionkit/client';
import type {MyApi} from './server.routes';

// AOT caches are loaded automatically
const {routes, hooks} = initClient<MyApi>({baseURL: 'http://localhost:3000'});
```

### Using Custom AOT Caches

For custom AOT packages, load them before initializing the client:

```ts
// Load custom AOT caches
import {jitFnsCache, pureFnsCache} from 'my-api-aot';
import {addAOTCaches} from '@mionkit/core';

addAOTCaches(jitFnsCache, pureFnsCache);

// Then initialize the client
import {initClient} from '@mionkit/client';
import type {MyApi} from './server.routes';

const {routes, hooks} = initClient<MyApi>({baseURL: 'http://localhost:3000'});
```

## Secure Environments (Cloudflare Workers, etc.)

AOT compilation is essential for running mion in secure environments that restrict dynamic code execution.

### Why AOT is Required

Environments like Cloudflare Workers, Deno Deploy, and some enterprise environments prohibit:
- `eval()`
- `new Function()`
- Other forms of dynamic code generation

mion's JIT compilation normally uses `new Function()` to generate validation and serialization functions. AOT compilation pre-generates these functions as static code, eliminating the need for runtime code generation.

### Cloudflare Workers Example

```ts
// worker.ts
import {routerCache, jitFnsCache, pureFnsCache} from 'my-api-aot';
import {addAOTCaches} from '@mionkit/core';

// Load AOT caches - no dynamic code generation needed
addAOTCaches(jitFnsCache, pureFnsCache);

import {initMionRouter} from '@mionkit/router';
import {routes} from './routes';

const myApi = initMionRouter(routes);

export default {
    async fetch(request: Request): Promise<Response> {
        // Handle request using mion router
        // ...
    },
};
```

## Verifying AOT Usage

To verify that AOT caches are being used correctly, you can check at runtime:

```ts
import {getJitUtils} from '@mionkit/core';

// After loading AOT caches and initializing routes
const jitUtils = getJitUtils();

// Check if a specific JIT function exists in cache
const cachedFn = jitUtils.getJIT('some-hash');
if (cachedFn) {
    console.log('Using pre-compiled JIT function');
}
```

## Cache Exports Reference

Each AOT package exports the following:

| Export | Type | Description |
|--------|------|-------------|
| `routerCache` | `MethodsCache` | Pre-compiled router method metadata |
| `jitFnsCache` | `PersistedJitFunctionsCache` | Pre-compiled JIT functions |
| `pureFnsCache` | `PersistedPureFunctionsCache` | Pre-compiled pure helper functions |

## Troubleshooting

### Caches Not Being Used

Ensure you call `addAOTCaches()` **before** any router or client initialization:

```ts
// ✅ Correct order
addAOTCaches(jitFnsCache, pureFnsCache);
const myApi = initMionRouter(routes);

// ❌ Wrong order - caches won't be used
const myApi = initMionRouter(routes);
addAOTCaches(jitFnsCache, pureFnsCache);
```

### Missing Functions in Cache

If some functions are missing from the AOT cache, ensure your build script includes all routes:

```bash
# Use the exact same entry point as production
npx mion-build-aot \
  --dir ./packages/my-api-aot \
  --start-server-script ./dist/cjs/server.js
```
